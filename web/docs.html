Coverage / Detectability Model

Overview

Coverage is computed as received power margin over a threshold: Rx(dBm) = EIRP - L_total, then Margin = Rx - Sensitivity. Cells with positive margin are detectable.

Path Loss Components

  • Free-space path loss (FSPL) using distance and frequency.
  • Terrain diffraction using Deygout multi-knife-edge over the DTM profile.
  • Foliage attenuation integrated along the path using canopy height (CHM).
  • Reliability margin: a simple fade margin added to total loss.

Reliability Slider

The UI reliability slider applies a fade margin from 0 dB at 50% to 20 dB at 99%. This is a pragmatic approximation, not a full ITU-R time/location availability model.

Detectability

The ELINT mode uses its own sensitivity threshold. By default it matches the device receiver sensitivity, but you can set it independently.

Device Defaults

Defaults are typical values and can be edited in the UI.

Note: Hunting 68–72 MHz and LV217 sensitivities use a typical narrowband FM receiver sensitivity (about 0.25 μV ≈ -119 dBm) as a proxy when device-specific specs are not available.

Performance Notes

The analysis grid controls compute cost. 256x256 is a good balance for browser-based diffraction + foliage integration. Larger grids increase compute time significantly.
